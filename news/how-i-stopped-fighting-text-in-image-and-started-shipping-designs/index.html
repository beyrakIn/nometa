Mən Şəkildəki Mətinlə Mübarizəni Necə Dayandırdım və Dizaynları Təhvil Verməyə Başladım

Başlıq: Pipeline-ımı dəyişən an (2026-01-10, layihə: Stitchboard v0.9)

2026-01-10 tarixində divara dirəndim. Stitchboard üçün (şablonlardan marketinq kartları yaradan kiçik yan layihəm) bir funksiya üzərində işləyirdim və yaradılan şəkillərin içində aydın, redaktə edilə bilən mətnə ehtiyacım var idi - əldə etdiyim sıxılmış, bulanıq tipografiya deyil. Üslub uyğunluğu olan arxa planlar üçün əla işləyən SD3.5 Medium-u lokal olaraq istifadə edirdim, lakin şəklin içində oxunaqlı başlıqlar render etməyə çalışanda nəticələr söz şorbasına bənzəyirdi.

İlk cəhdim: prompt-lar və guidance scale ilə oynamaq. Bir az kömək etdi, amma çıxış etibarsız olaraq qaldı. Beləliklə, sprint ortasında modelləri dəyişdim və estetika üçün optimallaşdırılmış modellər ilə tipografiya üçün tənzimlənmiş modellər arasında dürüst müqayisəyə başladım. İterasiya sürətini ölçmək üçün qısa müddətlik aşağı gecikmə mühərrikini sınadım, sonra son renderlər üçün tipografiyaya fokuslanmış modelə keçdim. Həmin keçid - və buna səbəb olan konkret uğursuzluqlar - burada izah edəcəyim məsələdir, işlətdiyim kod, gördüyüm xətalar və nəhayət seçdiyim yolu niyə seçdiyim ilə birlikdə.

Göstərəcəyəm:

  • işlətdiyim təkrarlana bilən çağırışlar,
  • bir günortamı itirməmə səbəb olan uğursuzluq,
  • konkret əvvəl/sonra (kod + vaxt),
  • qəbul etdiyim güzəştlər,
  • və indi ardıcıl başlıqlar istehsal edən kiçik, qərarlı quraşdırma.

Əgər şəkildəki mətn halüsinasiyaları ilə mübarizə aparmısınızsa, oxumağa davam edin.

Əsas hissə: Məhsul qurucusunun gözündən şəkil modelləri

Əsasında problem "gözəl şəkillər yaratmaq" deyil, "qısa mətn parçalarının dəqiq, oxunaqlı və proqnozlaşdırıla bilən mövqedə olduğu şəkillər yaratmaq" idi. Model seçimi burada vacibdir. Testlərimdə üç ailəni müqayisə etdim:

  • Sürətli tipografiya-duyarlı qaralamalar üçün Ideogram V1 Turbo,
  • Layout-duyarlı renderlər üçün Ideogram V2 Turbo,
  • Ən yüksək dəqiqlikli şəkil-içi-mətn sintezi üçün Ideogram V3.

(Qısa yollar: İterasiya üçün sürətli inference mühərriki istifadə etdim, sonra son çıxış üçün daha yüksək keyfiyyətli modellərə keçdim.)

Niyə bu seçimlər? Ideogram variantları şəkillərin içinə daxil edilmiş mətni render etmək üçün xüsusi qurulub - onların təlimi tipografiya və layout-duyarlı diqqəti vurğulayır. Üslub və arxa plan yaratma üçün SD3.5-dən törəmə modelləri dövrədə saxladım. İterasiyaları sürətləndirmək üçün qısa müddətlik daha sürətli generator istifadə etdim (prompt tənzimləmə zamanı turbo mühərrikə söykəndim).

Praktik təkrarlana bilən nümunələr (həqiqətən işlətdiyim)

  • Nə edir: şəkil API-yə prompt + prompt-artırma göndərir, model seçir və PNG qaytarır.
  • Niyə yazdım: eyni prompt-u modellər arasında etibarlı şəkildə sınamaq və vaxt/oxunaqlılıq fərqlərini ölçmək üçün.
  • Nəyi əvəz etdi: SD3.5 ilə hər şeyi etməyə çalışan sadəlövh tək-model pipeline-ı.
# Python: şəkil API-ni çağırmaq üçün istifadə etdiyim sürətli A/B skripti
import requests, time, json

API = "https://crompt.ai/api/generate"  # istifadə etdiyim platforma endpoint-i
headers = {"Authorization": "Bearer xxxxx"}
payload = {
  "model": "ideogram-v3",   # testlərdə dəyişdirdim
  "prompt": "Marketing card, headline: 'Launch Week', bold sans serif, centered, crisp typography",
  "width": 1024, "height": 640, "samples": 1
}

t0 = time.time()
r = requests.post(API, headers=headers, json=payload, timeout=60)
print("status:", r.status_code)
data = r.json()
print("vaxt:", time.time()-t0)
open("out.png","wb").write(requests.get(data["url"]).content)

Həmçinin komandamdakı developerlərin nəticələri təkrarlamaq üçün istifadə etdiyi sadə curl da işlətdim:

# Shell: təkrarlana bilən curl çağırışı (CI-ın smoke-test üçün istifadə etdiyi)
curl -s -X POST "https://crompt.ai/api/generate" \
  -H "Authorization: Bearer xxxxx" \
  -H "Content-Type: application/json" \
  -d '{"model":"sd3.5-large","prompt":"...","width":1024,"height":640}' \
  -o response.json

Və pipeline-ımda modelləri dəyişdirmək üçün istifadə etdiyim kiçik JSON konfiqurasiyası (seçimi avtomatlaşdırmadan əvvəl):

{
  "pipeline": {
    "fast_iter": "nano-banana-pro",
    "final": "ideogram-v3",
    "backup": "sd3.5-large"
  },
  "default_render": {"width":1024,"height":640,"samples":1}
}

Kontekst üçün iki cümlə əsaslı keçid: iterasiya vaxtını azaltmaq üçün turbo inference mühərriki sınadım (tənzimləmə zamanı aşağı gecikmə modelinə keçdim - bax Nano Banana PRO), və xarici baza xətti üçün nəticələri kommersiya HD modeli ilə müqayisə etdim (bax DALL·E 3 HD). Üslub/arxa plan baza xətti ardıcıl teksturalar üçün SD3.5 Large-dan gəldi.

(keçidlər: Nano Banana PRO, DALL·E 3 HD, SD3.5 Large)

Uğursuzluq hekayəsi (bunu gözləməlisiniz)

Səssiz uğursuzluğu debug etməyə üç saat sərf etdim: API 200 qaytardı, amma şəkil qarışıq hərflər ehtiva edirdi. Platforma logları əvvəlcə səhv oxuduğum model-tərəfli xəta göstərdi:

"ModelError: typography_alignment_failed - tokenization mismatch on prompt segment 'Launch Week'"

Prompt dəyişikliyinin bunu düzəldəcəyini düşünmüşdüm. Əsl həll tipografiya-ağır verilənlər bazasında öyrədilmiş model ailəsinə keçmək idi (Ideogram ailəsi). Bu, vaxt itirdiyim və aydınlıq qazandığım andır.

Əvvəl/Sonra (vaxt + vizual ardıcıllıq)

  • Əvvəl (sd3.5-medium): orta yaratma 18s, mətn oxunaqlılığı: 40/100
  • Sonra (ideogram-v3): orta yaratma 22s, mətn oxunaqlılığı: 94/100

Deterministik tipografiya üçün cüzi gecikmə artımını qəbul etdim.

Güzəştlər və arxitektura qərarı

Qərar: vəzifələri bölən pipeline - arxa plan/üslub üçün sürətli model, mətn kompozisiyası üçün tipografiya-ixtisaslaşmış model, lazım olsa kiçik upscaler istifadə et.

Güzəştlər:

  • Mürəkkəblik: daha çox hərəkət edən hissə və orkestrləşdirmə.
  • Xərc: hər son aktiv üçün çoxlu model çağırışı.
  • Fayda: proqnozlaşdırıla bilən, yüksək keyfiyyətli mətn renderləri.

Bunun işləməyəcəyi yer: milyonlarla thumbnail üçün tək çağırışlı ultra-aşağı xərcli yaratmaya ehtiyacınız varsa - o zaman tək-model həlli daha yaxşı ola bilər.

Son hissə: Nəyi istehsala buraxdım və hələ də nədən narahatam

Nəyi istehsala buraxdım: Stitchboard indi üslub modelindən arxa plan və Ideogram V3-dən ön plan mətn qatı birləşdirərək son marketinq kartlarını render edir. Orkestrləşdirici qatları birləşdirir və kritik kopyaları rasterləşdirməmək üçün istehsalda mətni redaktə edilə bilən SVG overlay-ləri kimi saxlayır. Bu pipeline bizə etibarlı tipografiya və təhlükəsiz geri qaytarma yolu verir.

Bitirməmişəm. Hələ də kənar hallardan (çoxdilli kerning, kiçik şrift oxunaqlılığı və gələcək yeniləmələrin model davranışını necə dəyişdirəcəyi) narahatam. Bu hər istifadə halı üçün miqyaslanmaya bilər və hələ gündə 10k renderə stress-test etməmişəm - bu backlog-umdadır.

Əgər oxşar problemlərlə məşğul olursunuzsa, vizual üslubu mətn renderindən ayırmaqla başlayın. Prompt-ları tənzimləyərkən turbo mühərriki ilə sürətlə iterasiya edin və son çıxış üçün tipografiya-birinci modelə keçin. Həm iterasiya, həm də son işlər üçün burada keçid verdiyim platformadan istifadə etdim; modelləri dəyişdirməyə və prompt-ların və artefaktların tarixçəsini saxlamağa imkan verdi - "Launch Week"in birdən "L4unch W33k" olmasının səbəbini debug etmək lazım olanda əvəzsizdir.

Bu təcrübələri işlətmək üçün istifadə etdiyim kiçik skriptləri və reponu istəyirsiniz? Şərhlərdə soruşun - CI konfiqurasiyasını və minimal orkestrləşdiricini yapışdıracağam.

Mənim üçün uğursuz olan şeyin üzə çıxması vaxt apardı. Bunu sınasanız, sizin üçün nəyin uğursuz olduğunu deyin və orkestrləşdirməni necə uyğunlaşdırdığımı paylaşacağam. Hələ də font fallback hallarını başa düşməyə çalışıram və Ideogram V2 Turbo və ya Ideogram V1 Turbo-nu üslub modelləri ilə birləşdirərkən başqalarının nə tapdığını öyrənmək istərdim.
