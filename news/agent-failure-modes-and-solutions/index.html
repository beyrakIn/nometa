Agent uğursuzluq rejimləri və həll yolları

Uzunmüddətli agentlər üçün effektiv idarəetmə strukturları. 26 Noyabr, 2025-ci ildə dərc edilib.

Agentlər hələ də bir çox kontekst pəncərəsi boyunca işləməkdə çətinliklərlə üzləşir. Uzunmüddətli agentlər üçün daha effektiv idarəetmə strukturu yaratmaqda ilham mənbəyi olaraq insan mühəndislərə müraciət etdik.

AI agentləri daha bacarıqlı olduqca, tərtibatçılar getdikcə daha çox onlardan saatlar, hətta günlər tələb edən mürəkkəb tapşırıqları yerinə yetirməyi istəyirlər. Lakin agentlərin bir neçə kontekst pəncərəsi boyunca ardıcıl irəliləyiş əldə etməsi hələ də həll olunmamış problemdir.

Uzunmüddətli agentlərin əsas problemi ondan ibarətdir ki, onlar diskret seanslar şəklində işləməli olurlar və hər yeni seans əvvəlki baş verənlər haqqında heç bir yaddaş olmadan başlayır. Növbəli işləyən mühəndislərlə dolu bir proqram layihəsi təsəvvür edin — hər yeni mühəndis əvvəlki növbədə nə baş verdiyini xatırlamadan gəlir.

Kontekst pəncərələri məhdud olduğundan və əksər mürəkkəb layihələr tək bir pəncərə daxilində tamamlana bilmədiyindən, agentlərin kodlaşdırma seansları arasındakı boşluğu aradan qaldırmaq üçün bir yola ehtiyacı var.

Biz Claude Agent SDK-nın bir çox kontekst pəncərəsi boyunca effektiv işləməsini təmin etmək üçün ikimərhələli həll hazırladıq: ilk işə salınmada mühiti quran başlatma agenti və hər seansta artımlı irəliləyiş edən, eyni zamanda növbəti seans üçün aydın artefaktlar buraxan kodlaşdırma agenti.

Kod nümunələrini əlavə edilmiş quickstart-da tapa bilərsiniz.

Uzunmüddətli agent problemi

Claude Agent SDK güclü, ümumi təyinatlı agent idarəetmə strukturudur — kodlaşdırmada, eləcə də modelin kontekst toplamaq, planlaşdırmaq və icra etmək üçün alətlərdən istifadə etməsini tələb edən digər tapşırıqlarda məharətlidir. O, kompaktlaşdırma kimi kontekst idarəetmə imkanlarına malikdir ki, bu da agentə kontekst pəncərəsini tükətmədən tapşırıq üzərində işləməyə imkan verir.

Nəzəri olaraq, bu quraşdırma ilə agentin ixtiyari uzun müddət faydalı iş görməyə davam etməsi mümkün olmalıdır. Lakin kompaktlaşdırma kifayət deyil.

Standart konfiqurasiyada, hətta Opus 4.5 kimi qabaqcıl kodlaşdırma modeli belə Claude Agent SDK üzərində bir neçə kontekst pəncərəsi boyunca dövrdə işlədildikdə, yalnız yüksək səviyyəli bir prompt verildiyi halda — məsələn, "claude.ai klonu qur" — istehsalata hazır keyfiyyətli veb tətbiqi qurmaqda uğursuz olacaq.

Claude-un uğursuzluqları iki nümunədə özünü göstərdi. Birincisi, agent həddən artıq çox işi birdəfəyə görməyə çalışırdı — mahiyyətcə tətbiqi bir cəhddə tamamlamağa çalışırdı. Bu, çox vaxt modelin tətbiqinin ortasında kontekstinin tükənməsinə gətirib çıxarırdı və növbəti seansı yarımçıq qalmış və sənədləşdirilməmiş funksionallıqla başlamağa məcbur edirdi. Agent sonra nə baş verdiyini təxmin etməli və əsas tətbiqin yenidən işlək vəziyyətə gətirilməsinə əhəmiyyətli vaxt sərf etməli olurdu. Bu, kompaktlaşdırma ilə belə baş verir, çünki kompaktlaşdırma həmişə növbəti agentə mükəmməl aydın təlimatlar ötürmür.

İkinci uğursuzluq rejimi adətən layihənin sonrakı mərhələlərində baş verirdi. Bəzi funksiyalar artıq qurulduqdan sonra, sonrakı agent nümunəsi ətrafına baxır, irəliləyişin olduğunu görür və işin bitdiyini elan edirdi.

Bu, problemi iki hissəyə ayırır. Birincisi, verilmiş prompt-un tələb etdiyi bütün funksiyalar üçün təməl yaradan ilkin mühit qurmalıyıq ki, agent addım-addım və funksiya-funksiya işləyə bilsin. İkincisi, hər agenti məqsədinə doğru artımlı irəliləyiş etməyə, eyni zamanda seansın sonunda mühiti təmiz vəziyyətdə buraxmağa yönəltməliyik. "Təmiz vəziyyət" dedikdə, əsas branch-ə birləşdirilmə üçün uyğun olan kod növünü nəzərdə tuturuq: böyük xətalar yoxdur, kod nizamlı və yaxşı sənədləşdirilmişdir və ümumiyyətlə, tərtibatçı əvvəlcə əlaqəsiz bir qarışıqlığı təmizləmədən yeni funksiya üzərində işə rahatlıqla başlaya bilər.

Daxili təcrübələrdə bu problemləri ikimərhələli həll yolu ilə aradan qaldırdıq:

Başlatma agenti: İlk agent seansı xüsusi prompt istifadə edir — modeldən ilkin mühiti qurmasını tələb edir: init.sh skripti, agentlərin nə etdiyini qeyd edən claude-progress.txt faylı və əlavə edilmiş faylları göstərən ilkin git commit.

Kodlaşdırma agenti: Hər sonrakı seans modeldən artımlı irəliləyiş etməsini, sonra strukturlaşdırılmış yeniliklər buraxmasını tələb edir.

Burada əsas kəşf, təzə kontekst pəncərəsi ilə başlayarkən agentlərin işin vəziyyətini tez anlaması üçün bir yol tapmaq idi — bu, git tarixçəsi ilə birlikdə claude-progress.txt faylı vasitəsilə həyata keçirilir. Bu təcrübələr üçün ilham effektiv proqram mühəndislərinin hər gün nə etdiyini bilməkdən gəldi.

Mühitin idarə edilməsi

Yenilənmiş Claude 4 prompt təlimatında biz çoxkontekstli iş axınları üçün bəzi ən yaxşı təcrübələri paylaşdıq, o cümlədən "ilk kontekst pəncərəsi üçün fərqli prompt" istifadə edən idarəetmə strukturu. Bu "fərqli prompt" başlatma agentindən gələcək kodlaşdırma agentlərinin effektiv işləməsi üçün lazım olan bütün kontekstlə mühiti qurmasını tələb edir.

Burada belə bir mühitin bəzi əsas komponentləri haqqında daha ətraflı məlumat veririk.
