Go-da Şimşək Sürətli ngrok Alternativi

Qısa Xülasə

Bir axşam ərzində (~3.5 saat konsentrasiyalı kodlaşdırma vaxtı) Go dilində tam ngrok-a bənzər tunel xidməti qurdum. Həm müştəri CLI, həm də backend server daxildir. Ümumi kod: ~800 sətir. Cloudflare Tunnels ilə işləyir və localhost-dan internetə pulsuz, təhlükəsiz HTTPS tunelləri təqdim edir.

Texnologiya Yığını: Go, Cloudflare Tunnels, Cloudflare API

Vebsayt: golocalport.link

İlham Mənbəyi

nport-u kəşf etdim - Node.js-də qurulmuş əla ngrok alternativi. Pulsuz, açıq mənbəlidir və Cloudflare infrastrukturundan istifadə edir. Amma mən bunları istəyirdim:

  • Daha kiçik həcm - Tək ikili fayl, Node.js runtime tələb olunmur
  • Daha sürətli başlanğıc - Go-nun kompilyasiya sürəti
  • Daha yaxşı paralellik - Nativ goroutine-lər
  • Öyrənmə imkanı - Tunel texnologiyasına dərin dalış

Niyə Go?

Xüsusiyyət Node.js (nport) Go (golocalport)
İkili faylın həcmi ~50MB + Node.js ~10MB müstəqil
Başlanğıc vaxtı ~500ms ~50ms
Yaddaş istifadəsi ~30MB ~5MB
Paralellik Event loop Nativ goroutine-lər
Asılılıqlar npm paketləri Minimal stdlib

Əsas Komponentlər

  1. CLI İnterfeysi - Flag təhlili, istifadəçi qarşılıqlı əlaqəsi
  2. API Müştərisi - Backend ilə əlaqə qurur
  3. İkili Fayl Meneceri - cloudflared-ı yükləyir/idarə edir
  4. Tunel Orkestratoru - Həyat dövrü idarəetməsi
  5. Vəziyyət Meneceri - Thread-safe icra vaxtı vəziyyəti
  6. UI Ekranı - Gözəl terminal çıxışı

İnkişaf Etdirmə Səyahəti

Mərhələ 1: Layihə Quraşdırması (15 dəqiqə)

Əsaslardan başladım:

go mod init github.com/devshark/golocalport

Təmiz layihə strukturu yaratdım:

golocalport/
├── cmd/golocalport/main.go       # Giriş nöqtəsi
├── internal/
│   ├── api/                 # Backend müştərisi
│   ├── binary/              # Cloudflared meneceri
│   ├── config/              # Konfiqurasiya
│   ├── state/               # Vəziyyət idarəetməsi
│   ├── tunnel/              # Orkestrator
│   └── ui/                  # Ekran
└── server/                  # Backend API

Əsas Dərslər

  1. Go-nun Stdlib-i Güclüdür

Bunlar üçün xarici asılılıq lazım deyil:

  • HTTP müştəri/server
  • JSON təhlili
  • Tar/gzip çıxarılması
  • Proses idarəetməsi
  • Siqnal emalı

  1. Cloudflare Tunnels Möhtəşəmdir

  • Pulsuz plan səxavətlidir
  • Qlobal edge şəbəkəsi
  • Avtomatik HTTPS
  • Port yönləndirmə lazım deyil
  • NAT/firewall arxasında işləyir

  1. Minimal Kod Daha Yaxşıdır

  • Dəstəkləmək asandır
  • Başa düşmək sürətlidir
  • Daha az səhv
  • Daha yaxşı performans

  1. Kross-Platforma Çətindir

Hər ƏS üçün fərqli ikili fayl formatları:

  • macOS: .tgz arxivi
  • Linux: xam ikili fayl
  • Windows: .exe

Həll: Runtime aşkarlaması + çıxarma məntiqi

Çətinliklər və Həllər

Çətinlik 1: İkili Fayl Format Fərqləri

Problem: macOS cloudflared xam ikili fayl deyil, .tgz formatındadır

Həll: Uzantını aşkarla, tar.gz-i anında çıxar

Çətinlik 2: Thread Təhlükəsizliyi

Problem: Birdən çox goroutine vəziyyətə daxil olur

Həll: Təhlükəsiz paralel giriş üçün sync.RWMutex

Çətinlik 3: Səliqəli Bağlanma

Problem: Ctrl+C-də təmizləmə

Həll: Siqnal emalı + defer cleanup

Çətinlik 4: Backend Hostinqi

Problem: Backend-i işlətmək üçün bir yer lazımdır

Həll: Çoxlu seçimlər - Fly.io (pulsuz), Railway, Docker, VPS

Nəticə

GoLocalPort-un qurulması əla öyrənmə təcrübəsi idi. Cəmi bir neçə saat ərzində belə bir istehsala hazır tunel xidməti yaratdım:

✅ macOS, Linux, Windows-da işləyir

✅ Sıfır xarici asılılıq

✅ Kiçik ikili fayl

✅ Anında başlayır

✅ Minimal yaddaş istifadə edir

✅ Həm müştəri, həm server daxildir

✅ Tam açıq mənbəlidir

Go bu tip sistem aləti üçün mükəmməl seçim olduğunu sübut etdi. Standart kitabxana lazım olan hər şeyi təqdim etdi və nəticədə yaranan ikili fayl kiçik, sürətli və portativdir.
