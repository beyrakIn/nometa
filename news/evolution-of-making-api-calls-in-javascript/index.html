JavaScript-də API Sorğularının Təkamülü

JavaScript-də API Çağırışlarının Təkamülü

Səhifəni yeniləmədən serverdən məlumat əldə etmək imkanı veb-i statik sənədlər kolleksiyasından bu gün istifadə etdiyimiz interaktiv "Web 2.0"-a çevirdi.

1. Erkən Dövr: XMLHTTPRequest (AJAX)

2000-ci illərin əvvəllərində Microsoft XMLHttpRequest (XHR) təqdim etdi. Bu, tərtibatçıların səhifə yükləndikdən sonra məlumat göndərib ala bildiyi ilk dəfə idi. Bu texnika AJAX (Asinxron JavaScript və XML) kimi tanındı.

Xüsusiyyətləri:

  • "Callback" əsaslı sistem istifadə edir.
  • Çox uzun və oxunması çətin.
  • Xətaların idarə edilməsi və çoxsaylı iç-içə sorğular "Callback Hell"-ə gətirib çıxarırdı.
// "Köhnə Məktəb" Üsulu
// IE5–IE6 stili (ActiveX)
var xhr = new ActiveXObject("Msxml2.XMLHTTP");
// və ya
var xhr = new ActiveXObject("Microsoft.XMLHTTP");

// Müasir brauzerlərarası (2006+)
var xhr = new XMLHttpRequest();

xhr.open('GET', 'https://api.example.com/users/123', true);  // true = asinxron

xhr.onload = function () {
  if (xhr.status >= 200 && xhr.status < 300) {
    var data = JSON.parse(xhr.responseText);
    console.log('Uğurlu:', data);
  } else {
    console.error('Sorğu uğursuz oldu. Status:', xhr.status);
  }
};

xhr.onerror = function () {
  console.error('Şəbəkə xətası');
};

xhr.send();   // sorğunu faktiki olaraq göndərir

Ağrılı nöqtələr → çoxlu boilerplate kod, manual status yoxlama, promise yoxdur.

2. Kitabxana Dövrü: jQuery AJAX

Orijinal XMLHttpRequest (XHR)-in uzunluq və brauzer uyğunsuzluğu problemlərini həll etmək üçün jQuery kimi kitabxanalar sənaye standartına çevrildi. O, sintaksisi əhəmiyyətli dərəcədə sadələşdirdi və brauzerlərarası uyğunluğu arxa planda idarə etdi.

Xüsusiyyətləri:

  • Daha təmiz, daha oxunaqlı kod: Xam XHR-dən qat-qat az boilerplate.
  • Sadələşdirilmiş xəta idarəsi: İnteqrasiya edilmiş success və error callback-ləri.
  • Callback əsaslı: Hələ də callback-lərə əsaslanırdı (sonrakı versiyalar deferred/promise təqdim edənə qədər).
// "jQuery" Üsulu
$.ajax({
    url: '[https://api.example.com/data](https://api.example.com/data)',
    method: 'GET',
    success: function(data) {
        console.log(data);
    },
    error: function(err) {
        console.error("Nəsə səhv getdi");
    }
});

→ Böyük məhsuldarlıq artımı, lakin hələ də callback əsaslı.

3. Axios – Promise əsaslı rahatlıq (2014+)

Fetch hər yerdə olmamışdan əvvəl çox populyar körpü idi.

axios.get("/api/users")
  .then(response => console.log(response.data))
  .catch(error => console.error(error.response?.data));

axios.post("/api/login", { email, password }, {
  headers: { Authorization: "Bearer token" }
});

Interceptor-lar, avtomatik JSON, daxili timeout kimi xüsusiyyətlər onu bir çox kod bazasında qalib etdi.

4. Fetch API – native müasir standart (2015 → indi)

Promise əsaslı, stream dəstəkli, əlavə kitabxana tələb etmir.

// Sadə GET
fetch("https://api.example.com/data")
  .then(response => {
    if (!response.ok) throw new Error(`HTTP ${response.status}`);
    return response.json();
  })
  .then(data => console.log(data))
  .catch(err => console.error(err));

// Müasir async/await stili (2017+)
async function getUsers() {
  try {
    const response = await fetch("/api/users", {
      method: "GET",
      headers: { "Accept": "application/json" },
      credentials: "include" // cookies və s.
    });

    if (!response.ok) {
      throw new Error(`Xəta ${response.status}`);
    }

    const data = await response.json();
    return data;
  } catch (err) {
    console.error("Fetch uğursuz oldu:", err);
  }
}

Fetch-in XHR üzərində təkmilləşdirmələri

  • Promise əsaslı → async/await ilə mükəmməl işləyir
  • Stream dəstəyi (response.body ReadableStream-dir)
  • Daha təmiz ləğvetmə (AbortController)
  • 404/500-də avtomatik JSON rejection yoxdur → response.ok yoxlamalısınız

Kod Oyun Meydançası

Cari vəziyyət (2026)

  • Fetch brauzerlərdə tövsiyə olunan native üsuldur (bütün müasir brauzerlərdə ~2017–2018-dən 100% dəstək)
  • Node.js Fetch-i v18-də (2022) stabil etdi → v21+-da defolt olaraq gəlir
  • Axios hələ də geniş istifadə olunur (xüsusilə interceptor-lar, daha yaxşı defoltlar lazım olan böyük kod bazalarında)
  • Xam XHR köhnəlmiş sayılır — yalnız çox xüsusi aşağı səviyyəli ehtiyaclar üçün istifadə olunur
